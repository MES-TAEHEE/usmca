기준정보 — 모든 모듈이 참조하는 공유 마스터 데이터 · 엔티티 카탈로그
The shared master data that every module references. Supplier, part, BOM, country, HTS, OEM customer, certificate type and label template form the common backbone consumed by DOC, LBL, ORG, AUD, INT and SYS. This volume is the entity catalog (L2 hub): it defines the 8 master entities, how they relate, and which modules consume them. Per-entity table-design L3 pages will follow as a later step — by work order, master-data L3 is done last, group by group.
모든 모듈이 참조하는 공유 마스터 데이터. 공급사·품목·BOM·국가·HTS·고객(OEM)·증빙 유형·라벨 템플릿은 DOC·LBL·ORG·AUD·INT·SYS가 함께 소비하는 공통 기반입니다. 본 권은 엔티티 카탈로그(L2 허브)로, 8개 마스터 엔티티와 상호 관계, 모듈별 소비 관계를 정의합니다. 엔티티별 테이블 설계 L3 페이지는 후속 단계에서 작성됩니다 — 작업 순서상 MD L3는 마지막에, 그룹별로 진행합니다.

BR-MD-01Change History & Approval
마스터 변경 이력·승인		모든 마스터 생성·수정·삭제는 이력 기록 + 승인 절차를 거침(누가·언제·무엇·이전값). 미승인 변경은 반영되지 않음.차단
BR-MD-02Code Values from SYS
SYS 공통코드 참조		구분·상태·유형 등 코드값은 SYS 공통코드를 참조. 자체 하드코딩 금지, 코드 변경은 SYS에서 일원화.검증
BR-MD-03Sync Source Priority
INT 동기화 소스 우선순위		INT 외부 동기화 시 소스 우선순위 적용(예: ERP > 수기). 충돌 시 우선 소스 값 채택, 하위 소스는 보류·로그.자동
BR-MD-04Referential Integrity
참조 무결성 (사용 중 삭제 차단)		다른 엔티티/모듈에서 참조 중인 마스터는 삭제 차단. 비활성(soft-delete)으로만 처리, 참조 해제 후 삭제 가능.차단
BR-MD-05Unique Required Keys
필수 키 유니크		각 엔티티의 필수 키(공급사코드·품번·국가코드·HTS·고객코드 등)는 유니크 제약. 중복 입력 시 저장 차단.차단

01
Overview
화면 개요
BOM Sync pulls the multi-level Bill of Materials from SAP S/4HANA into UOCM so the Origin (ORG) module can roll up component origins to the finished-part level. The screen shows each sync run with record counts (additions / changes / deletions), detected conflicts, and a per-run status (success / partial / failed). Sync executes on a daily batch schedule or on-demand via [Sync now]. When a run fails mid-apply, all changes are rolled back so UOCM never holds a half-applied BOM (BR-INT-01); rows with integrity errors are isolated and skipped rather than aborting the whole run (BR-INT-02). A change-preview panel lets the admin inspect new and modified BOM items before they propagate downstream.
BOM 동기화는 SAP S/4HANA의 다단계 자재명세서(BOM)를 UOCM으로 가져와, 원산지(ORG) 모듈이 구성품 원산지를 완성품 단위로 롤업할 수 있게 합니다. 화면은 각 동기화 실행을 건수(추가/변경/삭제)·충돌·실행상태(성공/부분/실패)와 함께 표시합니다. 동기화는 일배치 스케줄 또는 [지금 동기화]로 온디맨드 실행됩니다. 실행 중 실패 시 전체 변경을 롤백하여 반영 미완 BOM을 보관하지 않으며(BR-INT-01), 정합성 오류 행은 전체 중단 없이 격리·건너뜁니다(BR-INT-02). 변경 미리보기 패널로 다운스트림 전파 전 신규·변경 항목을 검토합니다.

04
Event & Action Flow
이벤트 · 액션 흐름
1
SCHEDBatch 02:00 / Click [Sync now]
일배치 스케줄러 또는 관리자의 [지금 동기화]로 동기화 잡 생성. int_job insert(상태 RUNNING).
진행 중 잡 있으면 신규 트리거 거부(중복 잠금, EXC-05).
RESULTSync job started · int_job = RUNNING
2
SYSTEMPull BOM from SAP
INT-01 연결로 SAP S/4HANA에서 다단계 BOM 추출(Delta 또는 Full). 스키마 검증.
연결 실패·스키마 변경 시 중단(→ EXC-05 / EXC-03).
RESULTBOM dataset staged · 원본 적재
3
SYSTEMDiff against md_bom
현행 md_bom과 비교하여 추가/변경/삭제 산출. 정합성 오류 행(orphan·순환·중복키) 격리(BR-INT-02).
변경 미리보기 패널에 결과 표시.
RESULTDiff = +126 / ~54 / −7 · 충돌 2 격리
4
SYSTEMApply (transactional)
단일 트랜잭션으로 md_bom upsert/delete 적용. 오류 발생 시 전체 롤백(BR-INT-01).
전 과정 int_job 로그 기록(BR-INT-04).
RESULTCommit = SUCCESS / PARTIAL · 또는 Rollback(→ EXC-01)
5
SYSTEMNotify + record run
실행 결과(성공/부분/실패)를 이력 그리드에 기록 + INT.Admin 알림. 격리 행은 INT-04로 전달.
부분 성공 시 충돌 건수와 사유 함께 통지.
RESULTRun logged · Admin notified
✓
SYSTEMFeed ORG rollup
반영된 BOM을 ORG-04 원산지 롤업이 참조. 구성품 원산지가 완성품 단위로 재계산됨(터미널).
UOCM BOM이 SAP와 정합 상태로 일치.
DONEBOM in sync · ORG rollup ready

Exception Handling
예외 처리
EXC-01Sync Failure → Rollback동기화 실패 롤백
Trigger적용 단계에서 오류 발생(DB·제약·중단).
Action트랜잭션 전체 롤백. md_bom 직전 상태 유지(BR-INT-01).
동기화 실패 — 변경 사항 롤백됨
EXC-02Integrity Error → Isolate정합성 오류 격리
Triggerorphan·순환참조·중복키 등 정합성 위반 행.
Action해당 행만 격리·건너뜀, 나머지 적용(BR-INT-02). 부분 성공.
2건 격리됨 — INT-04에서 확인하세요
EXC-03Schema Change스키마 변경
TriggerSAP BOM 구조·필드 변경으로 매핑 불일치(BR-INT-05).
Action적용 중단. 매핑 검토 요청 알림. 직전 상태 유지.
소스 스키마 변경 감지 — 매핑 확인 필요
EXC-04Partial Sync부분 동기화
Trigger일부 행 격리·일부만 적용 완료.
Action상태 "부분"으로 기록. 격리 건수·사유 통지.
부분 동기화 완료 — 충돌 2건 미반영
EXC-05Lock / Timeout잠금·타임아웃
Trigger진행 중 동기화 존재 또는 SAP 응답 지연.
Action신규 트리거 거부 또는 타임아웃 후 중단·재시도 안내.
진행 중 동기화가 있습니다 — 완료 후 재시도
EXC-06No Permission권한 없음
TriggerINT.Admin 미보유 사용자.
Action[지금 동기화] 비활성. 이력 열람만 가능.
동기화 실행 권한이 없습니다 (INT.Admin 필요)

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-INT-01
Rollback on failure — if any error occurs during apply, the entire sync transaction is rolled back; UOCM never holds a half-applied BOM.
적용 중 오류 발생 시 동기화 트랜잭션 전체 롤백. 반영 미완 BOM 보관 금지.
차단
BR-INT-02
Isolate integrity errors — rows failing integrity checks (orphan / cycle / duplicate key) are isolated and skipped; the run continues as a partial sync.
정합성 오류 행(orphan·순환·중복키)은 격리·건너뜀. 나머지는 부분 동기화로 진행.
대체
BR-INT-04
Full run logging — every sync run records source, type, counts, conflicts, duration, and outcome to int_job for audit and troubleshooting.
모든 동기화 실행의 소스·유형·건수·충돌·소요·결과를 int_job에 기록(감사·장애분석).

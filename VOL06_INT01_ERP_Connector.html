01
Overview
화면 개요
The ERP / SAP Connector is where IT/Integration Admins configure and monitor the single connection between UOCM and the enterprise backbone — SAP / ERP (and SCM). One screen defines the endpoint host, client, credentials, authentication method, runs a live connection test, and enables sync per business object (BOM, Supplier, PO) in either batch or real-time mode. This connection is the foundation for every downstream sync (INT-02/03) and integrity check (INT-04). Sensitive fields — passwords, tokens, secrets — are always masked on display and in logs (BR-INT-06), and any schema change on the ERP side is detected and surfaced (BR-INT-05) before it can corrupt a sync.
ERP/SAP 커넥터는 IT/연계 관리자가 UOCM과 전사 기간계 — SAP/ERP(및 SCM) — 간 단일 연결을 설정·모니터링하는 화면입니다. 한 화면에서 엔드포인트 호스트·클라이언트·자격증명·인증방식을 정의하고 실시간 연결 테스트를 수행하며, 업무객체(BOM·Supplier·PO)별로 배치 또는 실시간 동기화를 활성화합니다. 이 연결은 모든 하위 동기화(INT-02/03)와 정합성 검증(INT-04)의 기반입니다. 비밀번호·토큰·시크릿 등 민감정보는 화면·로그에서 항상 마스킹되며(BR-INT-06), ERP 측 스키마 변경은 동기화 손상 전에 감지·표시됩니다(BR-INT-05).

Exception Handling
예외 처리
EXC-01Connection / Auth Failure연결·인증 실패
Trigger연결 테스트 시 자격증명·인증서 불일치 또는 TLS 실패.
Action사유 코드 표시. 자동 재시도(backoff) 후 저장 차단.
연결 실패 — 자격증명·호스트 확인 후 재시도
EXC-02Endpoint Disabled엔드포인트 비활성
Trigger객체 상태 OFF 상태에서 동기화 트리거 발생.
Action해당 동기화 즉시 중단. OFF 사유 로그 기록.
엔드포인트 비활성 — 연계 건너뜀
EXC-03Credential Expired자격증명 만료
Trigger비밀번호·인증서·토큰 만료 또는 회전 누락.
Action연결 상태 경고. 갱신 안내 + 관리자 알림.
자격증명 만료 — Secret Vault에서 갱신 필요
EXC-04Schema Mismatch스키마 불일치
TriggerERP 측 필드·구조 변경 감지 (BR-INT-05).
Action연계 보류. 매핑 검토 요청 + INT-04 정합성 검증 연계.
스키마 변경 감지 — 매핑 검토 필요 (BR-INT-05)
EXC-05Timeout시간 초과
Trigger응답 지연이 임계치 초과 또는 네트워크 단절.
Actionbackoff 재시도 N회. 실패 지속 시 연결 DOWN 표시.
응답 시간 초과 — 재시도 중

TriggerINT.Admin 미보유 사용자 접근.
Action설정 폼·테스트·토글 비활성. 열람만 제한 또는 차단.

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-INT-06
Credential masking — passwords, tokens, and secrets are always masked on display and never written to logs; stored only in the SYS Secret Vault.
비밀번호·토큰·시크릿은 화면에서 항상 마스킹, 로그 기록 금지. SYS Secret Vault에만 저장.
차단
BR-INT-04
Integration logging — every connection test, config change, and endpoint toggle is logged (who / when / what) for audit and troubleshooting.
연결 테스트·설정 변경·엔드포인트 토글 전수 로그 기록(감사·장애 추적).
자동
BR-INT-05
Schema-change guard — any detected ERP-side schema change pauses affected syncs and requires mapping review before resuming.
ERP 측 스키마 변경 감지 시 해당 연계 보류, 매핑 검토 후 재개.

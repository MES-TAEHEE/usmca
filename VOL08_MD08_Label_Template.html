01
Entity Overview
엔티티 개요
The Label Template entity defines reusable label layout definitions per OEM and country-of-origin (COO) form used by the Label module (LBL). Each template is a parent record (md_label_template) carrying paper size, barcode type, version and approval status, plus a set of child field rows (md_label_template_field) that map each printable field — origin country, part, lot, certificate no., serial — to a data source and a physical position (X / Y) on the label. Templates are approval-gated: only an approved template (BR-LBL-02) may be selected at print time, guaranteeing every issued label conforms to a controlled, versioned layout. Standard COO forms are OEM-agnostic; OEM-specific templates carry an oem_id and may enforce additional required fields (BR-LBL-05).
라벨 템플릿 엔티티는 라벨 모듈(LBL)에서 사용하는 OEM·원산지(COO) 양식별 재사용 라벨 레이아웃을 정의합니다. 각 템플릿은 용지 규격·바코드 종류·버전·승인상태를 담는 부모 레코드(md_label_template)와, 인쇄 필드(원산지국·품목·LOT·인증서번호·시리얼)를 데이터 소스와 라벨상의 물리 좌표(X/Y)에 매핑하는 자식 필드 행(md_label_template_field)으로 구성됩니다. 템플릿은 승인 게이트 대상으로, 발행 시 승인됨 템플릿만 선택 가능(BR-LBL-02)하여 모든 발행 라벨이 통제·버전관리된 레이아웃을 따르도록 보장합니다. COO 표준 양식은 OEM 무관이며, OEM 전용 템플릿은 oem_id를 가지고 추가 필수 필드를 강제할 수 있습니다(BR-LBL-05).

06
Business Rules
업무 규칙
IDRule · 규칙Action
BR-MD-01
Master data integrity — required attributes (code, name, label type, paper/barcode) must be present before a template can be saved.
기준정보 무결성 — 코드·명칭·라벨유형·용지/바코드 등 필수 속성 입력 후에만 저장 가능.
차단
BR-MD-04
Unique key — template_code + version must be unique; the same code may exist only once per version.
유니크 키 — template_code + version 조합은 유일해야 하며 동일 코드는 버전당 1건만 허용.
차단
BR-MD-05
Soft delete — templates are never physically removed; is_active=0 retires a template while preserving issued-label references.
논리 삭제 — 템플릿은 물리 삭제하지 않고 is_active=0 으로 비활성화, 발행 라벨 참조 보존.
자동
BR-LBL-02
Approval-gated use — only approval_status='approved' templates are selectable at print time; draft templates are hidden from issuance.
승인 게이트 — 발행 시 approval_status='approved' 템플릿만 선택 가능, draft는 발행 목록에서 제외.
차단
BR-LBL-05
OEM required fields — OEM templates may flag fields as required (is_required=1); issuance is blocked if any required field has no value.
OEM 필수 필드 — OEM 템플릿은 필드를 필수(is_required=1)로 지정 가능, 필수 값 누락 시 발행 차단.

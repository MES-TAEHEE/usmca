01
Overview
화면 개요
Notification Settings is the central console where the administrator configures a notification rule per event type — certificate expiry (D-90 / D-30 / D-7), approval pending, sync failure, integrity error — and maps each to delivery channels (Email / SMS / Push / 사내 메신저) and recipient roles. A rule grid lists every event with its channel badges, recipient roles, retry policy and an enable toggle; a right-hand channel-config panel holds the connection credentials for each channel (SMTP, SMS gateway, push key, messenger webhook — all secrets masked) plus a test-send button. Failed deliveries follow the retry-then-escalate policy (BR-SYS-04), and every send is logged (BR-SYS-02). This screen is the engine that drives DOC-03 expiry alerts, DOC-04 approval prompts and INT-04 integrity warnings across all modules.
알림 설정은 관리자가 이벤트 유형별 알림 규칙을 정의하는 중앙 콘솔입니다 — 증빙 만료(D-90/D-30/D-7), 승인 대기, 동기화 실패, 무결성 오류. 각 이벤트를 전송 채널(Email/SMS/Push/사내 메신저) 및 수신 역할에 매핑합니다. 규칙 그리드는 채널 배지·수신 역할·재시도 정책·활성 토글을 표시하고, 우측 채널 설정 패널은 채널별 연결 정보(SMTP·SMS 게이트웨이·푸시 키·메신저 웹훅 — 시크릿 마스킹)와 테스트 발송 버튼을 제공합니다. 전송 실패는 재시도 후 에스컬레이션 정책(BR-SYS-04)을 따르며, 모든 발송은 로깅됩니다(BR-SYS-02). 본 화면은 DOC-03 만료 알림, DOC-04 승인 알림, INT-04 무결성 경고를 전 모듈에서 구동하는 엔진입니다.

Exception Handling
예외 처리
EXC-01Send Failure전송 실패
Trigger채널 발송 실패(SMTP 거부·게이트웨이 오류 등).
Action설정 횟수·간격 재시도 → 소진 시 SYS.Admin 에스컬레이션 (BR-SYS-04). 실패 로그.
발송 실패 — 재시도 후 에스컬레이션됨
EXC-02Channel Misconfigured채널 미설정
Trigger연결 정보 누락·오류 채널을 규칙에 매핑.
Action규칙 저장 차단. 채널 설정 패널로 이동 안내.
채널 설정을 먼저 완료하세요
EXC-03No Recipient수신자 없음
Trigger매핑 역할에 활성 사용자 0명.
Action기본 수신자(SYS.Admin)로 폴백 발송 + 경고.
수신자 없음 — 기본 관리자에게 발송
EXC-04Rate Limit발송 한도 초과
Trigger채널/게이트웨이 발송 한도 초과.
Action큐 보류 후 지연 발송. 묶음(digest) 처리 제안.
한도 초과 — 지연 발송으로 전환
EXC-05Masked Secret Edit마스킹 시크릿 수정
Trigger마스킹된 시크릿 필드 편집 시도.
Action기존 값 미노출. 신규 값 입력 시에만 교체(쓰기 전용).
시크릿은 표시되지 않음 — 새 값 입력 시 교체
EXC-06No Permission권한 없음
TriggerSYS.Admin 미보유 사용자 접근.
Action화면 진입 차단·읽기 전용. 편집 버튼 비활성.
설정 권한이 없습니다 (SYS.Admin 필요)

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-SYS-04
Retry & escalation — failed deliveries retry per configured count/interval; on exhaustion, escalate to SYS.Admin (or the rule's escalation role).
전송 실패 시 설정 횟수·간격으로 재시도, 소진 시 SYS.Admin(또는 지정 상위 역할)에게 에스컬레이션.
자동
BR-SYS-02
All sends logged — every notification dispatch (recipient, channel, time, result) and every rule change is recorded for audit (visible in SYS-02).
모든 알림 발송(수신자·채널·시각·결과)과 규칙 변경은 감사용으로 기록(SYS-02 조회).
자동
BR-SYS-01
Admin-only configuration — only SYS.Admin may create, edit, or toggle notification rules and channel credentials; others are read-only/blocked.
알림 규칙·채널 자격증명의 생성·편집·토글은 SYS.Admin만 가능, 그 외 읽기 전용·차단.

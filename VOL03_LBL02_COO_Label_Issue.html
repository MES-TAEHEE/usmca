01
Overview
화면 개요
The COO Label Issue screen issues Certificate-of-Origin labels for parts and shipments. When a part is selected, the system verifies that it has a VALID certificate in DOC — an expired or missing certificate blocks issuance (BR-LBL-01) — and pulls the origin country from ORG. On issue, the system assigns a unique serial number (BR-LBL-06), renders the label from the approved template, queues it to print (LBL-05), and writes an immutable record to the issuance history (LBL-04, BR-LBL-03). This screen guarantees that every physical label shipped is backed by valid, traceable origin evidence.
원산지 라벨 발행 화면은 품목·출하건에 대한 원산지증명(COO) 라벨을 발행합니다. 품목 선택 시 시스템은 해당 품목이 DOC에 유효한 증빙을 보유하는지 검증하며, 만료·부재 증빙은 발행을 차단합니다(BR-LBL-01). 원산지국은 ORG에서 자동으로 가져옵니다. 발행 시 고유 일련번호 부여(BR-LBL-06), 승인 템플릿 기반 렌더링, 출력 큐 등록(LBL-05), 발행 이력 불변 기록(LBL-04, BR-LBL-03)을 수행합니다. 출하되는 모든 물리 라벨이 유효·추적 가능한 원산지 증빙에 근거함을 보장합니다.

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-LBL-01
Valid evidence required — a label cannot be issued unless the part has a VALID (non-expired, approved) certificate in DOC. Expired or missing evidence blocks issuance.
발행 전 품목에 유효(미만료·승인)한 DOC 증빙이 있어야 발행 가능. 만료·부재 증빙은 발행 차단.
차단
BR-LBL-06
Serial uniqueness — every issued label receives a system-generated unique serial number; collisions auto-retry with the next sequence value.
모든 발행 라벨에 시스템 채번 고유 일련번호 부여. 충돌 시 다음 번호로 자동 재시도.
자동
BR-LBL-03
Issuance immutable — once issued, an issuance record cannot be edited or deleted; corrections require a new issuance (void + reissue) for full audit trail.
발행 건은 수정·삭제 불가. 정정은 신규 발행(취소+재발행)으로 처리, 전 과정 감사 추적 보존.

/ ADMIN
시스템 관리 — 사용자·권한(RBAC) · 시스템 감사 로그 · 알림 설정 · 공통 코드 관리
The cross-cutting platform foundation that underpins every module. Establishes RBAC users & roles (least-privilege access to DOC / LBL / ORG / AUD / INT), an immutable system audit log capturing all login, access, and change events for security and compliance, notification settings that govern alert channels for expiry / approval / integration-sync events, and common-code management referenced by all modules. Not a business workflow — it is the governance, security, and configuration layer the whole platform runs on.
모든 모듈을 떠받치는 플랫폼 공통 토대. 최소권한 원칙의 사용자·권한(RBAC)(DOC / LBL / ORG / AUD / INT 접근 제어), 로그인·접근·변경 이벤트를 보안·컴플라이언스용으로 기록하는 불변 시스템 감사 로그, 만료·승인·연동 동기화 이벤트의 알림 채널을 관장하는 알림 설정, 전 모듈이 참조하는 공통 코드 관리를 구축합니다. 업무 워크플로우가 아니라 플랫폼 전체가 의존하는 거버넌스·보안·설정 계층입니다.

SYS-02
Audit Trail
시스템 감사 로그 — 로그인·접근·변경 불변 기록
🖥 Web PCPhase 1
📋 L3 Detail →
👀
What you'll see: 보안 이벤트 타임라인(로그인·접근·변경) + 사용자·모듈·시점 필터 + 불변 로그 검색·내보내기모든 보안 이벤트를 변경 불가 형태로 보존 — 컴플라이언스 대응
EN · Description
Immutable system-wide audit log. Records every login, access, and change event (who / when / what / from where) across all modules. Append-only — entries cannot be edited or deleted. Filter by user, module, event type, and time; export for compliance/security review. Distinct from AUD (business audit support).
KO · 설명
전 시스템 불변 감사 로그. 모든 모듈의 로그인·접근·변경 이벤트(누가/언제/무엇/어디서)를 기록. 추가 전용(append-only) — 항목 수정·삭제 불가. 사용자·모듈·이벤트유형·시점으로 필터, 컴플라이언스·보안 검토용 내보내기. AUD(업무 감사 지원)와는 별개의 시스템 보안 로그.

SYS-03
Notification Settings
알림 설정 — 만료·승인·동기화 이벤트 채널 관리
🖥 Web PCPhase 1
📋 L3 Detail →
👀
What you'll see: 이벤트별 알림 매핑(만료/승인/동기화) + 채널 선택(Email/SMS/Push/메신저) + 재시도·에스컬레이션 정책어떤 이벤트를 누구에게 어느 채널로 — 전 모듈 알림을 한 곳에서 설정
EN · Description
Central notification configuration for the whole platform. Maps event types (certificate expiry, approval requests, integration-sync failures) to recipients and channels (Email / SMS / Push / in-house messenger), with thresholds, retry, and escalation policies. Modules raise events; SYS-03 governs how and to whom they are delivered.
KO · 설명
플랫폼 전체 알림 중앙 설정. 이벤트유형(증빙 만료, 승인 요청, 연동 동기화 실패)을 수신자·채널(Email / SMS / Push / 사내 메신저)에 매핑하고, 임계값·재시도·에스컬레이션 정책 관리. 모듈이 이벤트를 발생시키면 SYS-03이 전달 방식·대상을 관장.

SYS-04
Code Management
공통 코드 관리 — 전 모듈 참조 코드 그룹
🖥 Web PCPhase 1
📋 L3 Detail →
👀
What you'll see: 코드 그룹·코드값 트리 + 다국어 명칭 + 사용처(모듈) 표시 + 변경 영향 검증전 모듈이 참조하는 공통 코드를 한 곳에서 정의·관리
EN · Description
Single registry of common codes (code groups + code values) referenced by every module — statuses, types, units, countries, reason codes, etc. Supports multilingual labels, sort order, active/inactive, and usage references. Changing a code that modules depend on triggers an impact check before save (BR-SYS-05).
KO · 설명
전 모듈이 참조하는 공통 코드(코드 그룹 + 코드값)의 단일 등록부 — 상태·유형·단위·국가·사유 코드 등. 다국어 명칭, 정렬 순서, 활성/비활성, 사용처 참조 지원. 모듈이 의존하는 코드를 변경하면 저장 전 영향 검증 수행(BR-SYS-05).

BR-SYS-06Session & Password Policy
세션·비밀번호 정책		세션 만료·자동 로그아웃, 비밀번호 복잡도·주기적 변경, 로그인 실패 한도 초과 시 계정 잠금(접근 차단).차단

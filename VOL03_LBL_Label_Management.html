MANAGEMENT
라벨 관리 — 원산지(COO) 라벨 · OEM 고객별 양식 발행 · 출력 · 발행 이력
Phase 2 · operated directly by the corporation. Issues origin (COO) labels and OEM customer-specific labels (현대/기아/MOBIS) from approved templates. Labels may ONLY be issued against a VALID certificate in DOC — if the supporting evidence is expired, issuance is blocked (BR-LBL-01). Validates OEM form-specific required fields, assigns unique serial numbers, supports reprints with logged reasons, and keeps a full, immutable issuance history. Consumes DOC (valid certificate) and ORG (origin data).
Phase 2 · 법인 직접 운영. 승인된 템플릿으로 원산지(COO) 라벨과 OEM 고객별 양식 라벨(현대/기아/MOBIS)을 발행합니다. 라벨은 DOC의 유효 인증서가 있을 때만 발행 가능하며, 근거 증빙이 만료된 경우 발행이 차단됩니다(BR-LBL-01). OEM 양식별 필수필드를 검증하고, 유니크 일련번호를 부여하며, 사유 기록과 함께 재발행을 지원하고, 불변 발행 이력 전체를 유지합니다. DOC(유효 인증서)·ORG(원산지 데이터)를 활용합니다.

EN · Description
Visual designer for label templates. Define layout, fixed text, dynamic fields (origin country, part, LOT, certificate no., serial), barcodes and QR codes. Maintain separate templates per OEM form (COO standard, 현대/기아/MOBIS). Templates pass an approval step before they can be used for issuance.

EN · Description
Core issuance screen for origin (COO) labels. On part/LOT selection the system checks DOC for a valid certificate and pulls origin data from ORG. A unique serial is assigned, the label is previewed against the approved template, and issuance is recorded. If the supporting evidence is expired or missing, issuance is hard-blocked (BR-LBL-01).
KO · 설명
원산지(COO) 라벨 발행 핵심 화면. 품목/LOT 선택 시 DOC의 유효 인증서를 확인하고 ORG에서 원산지 데이터를 가져옴. 유니크 일련번호 부여 후 승인 템플릿으로 미리보기·발행하고 이력 기록. 근거 증빙이 만료·누락이면 발행 강제 차단(BR-LBL-01).

EN · Description
Issues OEM customer-specific labels per each customer's required form (현대 HMC, 기아 KIA, MOBIS). Issuance History
발행 이력 — 불변 이력 · 취소만 가능 · 재발행 사유 추적
🖥 Web PCPhase 2
📋 L3 Detail →
👀
What you'll see: 발행 이력 그리드(일련번호·품목·LOT·고객·증빙·발행일·발행자) + 취소/재발행 추적발행 이력은 불변, 수정 불가 — 취소만 가능 (BR-LBL-03)
EN · Description
Full, immutable record of every issued label. Grid shows serial, part, LOT, customer/form, referenced certificate, issue date and issuer. Records cannot be edited — only cancelled, and reprints capture a logged reason (BR-LBL-04). Feeds AUD with a defensible issuance trail.
KO · 설명
발행된 모든 라벨의 불변 이력. 그리드에 일련번호·품목·LOT·고객/양식·참조 증빙·발행일·발행자 표시. 이력은 수정 불가, 취소만 가능하며 재발행 시 사유 기록(BR-LBL-04). AUD에 방어 가능한 발행 추적 제공.

▼ BUSINESS RULES · 업무 규칙
IDRule · 규칙Detail · 상세Action
BR-LBL-01Valid Evidence Required
유효 증빙 필수		유효한 DOC 증빙이 없거나 만료된 경우 라벨 발행 차단. DOC 만료 상태와 실시간 연계.차단
BR-LBL-02Approved Template Only
템플릿 승인 후 사용		승인되지 않은 템플릿으로는 발행 불가. 템플릿은 승인 단계를 거친 후에만 발행에 사용.차단
BR-LBL-03Immutable History
발행 이력 불변		발행 이력은 수정·삭제 불가. 잘못된 발행은 취소(cancel)로만 처리, 취소 이력도 보존.차단
BR-LBL-04Reprint Reason Logged
재발행 사유 기록		재발행/재출력 시 사유 입력 필수. 원 발행 건 참조와 함께 사유 로그 기록.검증
BR-LBL-05OEM Field Validation
OEM 양식별 필수필드 검증		고객 양식(현대/기아/MOBIS)별 필수필드를 모두 충족해야 발행. 누락 시 발행 차단.차단
BR-LBL-06Unique Serial
일련번호 유니크		라벨 일련번호는 전 발행 건에서 유니크. 중복 일련번호 발행 불가.차단

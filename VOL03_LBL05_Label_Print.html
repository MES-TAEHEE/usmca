01
Overview
화면 개요
The Label Print screen renders issued origin labels to a physical label printer (Zebra ZPL) by template. A Web print queue lists every issued-but-unprinted label and supports per-label print, batch print to a selected printer, and template-driven layout. At the shipping floor, a PDA / handheld reprint flow lets operators recover from physical damage: scan the label serial → preview → reprint on the nearest printer. Every print and reprint increments the print count and is recorded in the immutable history (LBL-04); reprints require a reason (BR-LBL-04). Tables: lbl_issue, lbl_print_job.
라벨 출력 화면은 발행된 원산지 라벨을 템플릿별로 물리 라벨 프린터(Zebra ZPL)로 출력합니다. Web 출력 큐에 미출력 라벨을 표시하고 개별 출력·일괄 출력(프린터 선택)·템플릿 기반 레이아웃을 지원합니다. 출하 현장에서는 PDA 재출력 흐름으로 손상 라벨을 복구합니다: 일련번호 스캔 → 미리보기 → 재출력. 모든 출력·재출력은 출력 횟수를 증가시키고 변경 불가 이력(LBL-04)에 기록되며, 재출력 시 사유 입력이 필요합니다(BR-LBL-04). 테이블: lbl_issue, lbl_print_job.

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-LBL-04
Reprint reason required — every reprint must capture a reason (damaged / lost / unreadable / other) before printing; logged to LBL-04.
재출력 시 사유(손상/분실/판독불가/기타) 필수 입력 후에만 출력 · LBL-04 기록.
차단
BR-LBL-06
Serial on label — every printed label carries a unique serial number (barcode); the same serial is reused on reprint, never regenerated.
모든 라벨에 고유 일련번호(바코드) 인쇄 · 재출력 시 동일 일련번호 재사용, 신규 생성 금지.
자동
BR-LBL-03
History immutable — print/reprint history records and print counts cannot be edited or deleted; corrections are append-only.
출력·재출력 이력 및 출력 횟수는 수정·삭제 불가 · 정정은 추가 기록만 허용.

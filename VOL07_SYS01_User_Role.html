01
Overview
화면 개요
User & Role (RBAC) is the central console where the System Admin manages all accounts and role-based permissions on a least-privilege basis (BR-SYS-01). A user grid lists every account with assigned roles, status, and last login; a role-permission matrix grants module-scoped capabilities (DOC / LBL / ORG / AUD / INT / SYS) per role. Users are assigned one or more roles, never raw permissions. Any change to permissions or role mapping requires approval and is fully logged (BR-SYS-03), and password / session policy is enforced on every account (BR-SYS-06). SSO accounts are linked through the INT module. Tables: sys_user, sys_role, sys_role_permission.
사용자·권한(RBAC)은 시스템 관리자가 최소 권한 원칙(BR-SYS-01)에 따라 모든 계정과 역할 기반 권한을 관리하는 중앙 콘솔입니다. 사용자 그리드는 역할·상태·최종 로그인을 표시하며, 역할-권한 매트릭스로 모듈별 권한(DOC/LBL/ORG/AUD/INT/SYS)을 역할 단위로 부여합니다. 사용자는 개별 권한이 아닌 역할을 통해서만 권한을 받습니다. 권한·역할 매핑 변경은 승인 후 적용되며 전수 로그됩니다(BR-SYS-03). 비밀번호·세션 정책은 모든 계정에 적용되며(BR-SYS-06), SSO 계정은 INT 모듈로 연동됩니다. 테이블: sys_user, sys_role, sys_role_permission.

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-SYS-01
Least privilege — users receive only the minimum roles needed; over-privileged role combinations raise a warning requiring justification.
최소 권한 원칙 — 필요한 최소 역할만 부여. 과다 권한 조합은 사유 입력을 요구하는 경고 발생.
경고
BR-SYS-03
Change approval & log — every permission / role-mapping change requires approval before it takes effect and is recorded immutably (who / when / before→after).
변경 승인·로그 — 모든 권한·역할 매핑 변경은 승인 후 적용되며 불변 로그(누가·언제·이전→이후)로 기록.
차단
BR-SYS-06
Session & password policy — enforce password length/complexity/reuse rules, forced reset on temp passwords, and session timeout on all accounts.
세션·비밀번호 정책 — 길이·복잡도·재사용 규칙, 임시 비밀번호 변경 강제, 세션 만료를 전 계정에 적용.

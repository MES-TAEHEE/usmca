01
Entity Overview
엔티티 개요
The HTS Code entity is the harmonized tariff classification reference for the entire UOCM platform. Each record pairs an HTS heading with its tariff rates (base duty + USMCA preferential rate) and the USMCA origin rule applicable to that heading — the specific tariff shift or Regional Value Content (RVC) requirement that determines whether a good qualifies as originating. As such, md_hts serves a dual purpose: it is the basis for duty calculation and the rule source for origin determination. Parts (MD-02) reference an HTS code, the origin engine (ORG-05) reads the heading's origin_rule and rvc_threshold to drive its qualification logic, and certificates (DOC) classify HTS-type evidence against it.
HTS Code 엔티티는 UOCM 플랫폼 전체의 관세 품목분류 기준정보입니다. 각 레코드는 HTS 호(heading)에 대해 관세율(기본세율 + USMCA 특혜세율)과 해당 호에 적용되는 USMCA 원산지 규칙(원산지 결정을 좌우하는 세번변경(tariff shift) 또는 역내부가가치(RVC) 요건)을 함께 보유합니다. 따라서 md_hts는 관세 산정의 기초이자 원산지 판정의 규칙 원천이라는 이중 역할을 합니다. 품목(MD-02)이 HTS 코드를 참조하고, 원산지 엔진(ORG-05)은 호의 origin_rule·rvc_threshold를 읽어 판정 로직을 구동하며, 증빙(DOC)은 HTS 유형 증빙을 이에 대조합니다.

06
Business Rules
업무 규칙
IDRule · 규칙Action
BR-MD-01
Master integrity — every master record carries created/updated audit columns; reference masters are not hard-deleted.
모든 기준정보는 생성·수정 감사 컬럼을 보유하며, 참조 기준정보는 물리 삭제하지 않음.
자동
BR-MD-02
Logical deactivation — deprecated HTS codes set is_active = 0; referenced codes cannot be removed, only deactivated.
폐지 HTS 코드는 is_active = 0 처리. 참조 중인 코드는 삭제 불가, 비활성만 가능.
차단
BR-MD-04
Unique HTS code — hts_code must be unique across the table; duplicate insert is rejected.
hts_code 유니크 — 동일 코드 중복 등록 거부.
차단
BR-MD-05
Origin rule drives ORG — origin_rule and rvc_threshold are the rule source for ORG-05 origin/RVC determination; rates must be ≥ 0.
origin_rule·rvc_threshold가 ORG-05 원산지·RVC 판정의 규칙 원천. 세율은 0 이상.

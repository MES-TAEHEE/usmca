01
Overview
화면 개요
The Version History screen shows the immutable version chain for a single certificate. Each renewal or replacement creates a new version while every prior version stays viewable — capturing who changed it, when, and why. This eliminates the shared-drive "version conflict" problem where the latest file silently overwrites the original. Superseded versions are clearly flagged, the current version is always identifiable, and nothing is ever deleted (BR-DOC-05). A side-by-side compare lets auditors see exactly what changed between any two versions (e.g. an extended expiry date). The screen is read-only: there is no restore or edit — immutability is the guarantee.
버전 이력 화면은 단일 증빙의 불변 버전 체인을 보여줍니다. 갱신·교체 시마다 신규 버전이 생성되고, 이전 버전은 모두 열람 가능하게 보존되어 누가·언제·왜 변경했는지를 기록합니다. 최신 파일이 원본을 덮어쓰는 공유폴더 "버전 충돌"을 근본 제거합니다. 대체된 버전은 명확히 표시되고, 현행 버전은 항상 식별 가능하며, 어떤 버전도 삭제되지 않습니다(BR-DOC-05). 나란히 비교(Compare)로 두 버전 간 변경점(예: 만료일 연장)을 정확히 확인할 수 있습니다. 본 화면은 읽기 전용 — 복원·편집은 없으며 불변성이 보증됩니다.

Exception Handling
예외 처리
EXC-01Delete Prohibited삭제 금지
Trigger버전 삭제 시도 (API·직접 요청 포함).
Action차단 (BR-DOC-05). 삭제 메뉴 미노출 + 서버 거부.
버전은 삭제할 수 없습니다 · Versions are immutable
EXC-02Restore Prohibited복원 금지
Trigger이전 버전으로 되돌리기(restore) 시도.
Action복원 버튼 항상 비활성. 정식 갱신(신규 버전) 안내.
복원 불가 — 갱신은 신규 버전으로 처리
EXC-03Single Version단일 버전
Trigger버전이 1개뿐 또는 동일 버전 2개 선택.
Action[⇄ Compare] 비활성. 비교 대상 부족 안내.
비교하려면 서로 다른 2개 버전이 필요합니다
EXC-04Missing File파일 누락
Trigger구버전 원본 파일 조회 실패(저장소 누락).
Action메타는 표시, 파일 열람만 차단. 관리자 알림.
해당 버전 파일을 찾을 수 없습니다 — 메타만 표시
EXC-05No Permission권한 없음
TriggerDOC.View 미보유 사용자 접근.
Action화면 진입 차단. 열람 권한 안내.
열람 권한이 없습니다 (DOC.View 필요)
EXC-06Concurrent Renewal동시 갱신 충돌
Trigger동일 증빙에 두 갱신이 동시 승인 시도.
Action순차 처리(낙관적 락). 후행 건은 다음 버전으로 적재.
동시 갱신 감지 — 순차 버전으로 처리되었습니다

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-DOC-05
Immutable versioning — approved certificates are never deleted or overwritten; each renewal/replacement creates a new version, prior versions preserved and the latest is flagged current.
불변 버전 관리 — 승인 증빙은 삭제·덮어쓰기 불가. 갱신·교체 시 신규 버전 생성, 이전 버전 보존·최신본 현행 표시.
차단
BR-DOC-06
Audit trail — every version creation, view, and compare records who / when / why for audit retention.
감사 추적 — 버전 생성·열람·비교 시 누가·언제·왜를 감사 보존용으로 전수 기록.
자동
BR-DOC-02
Only the approved version is valid — the current (latest approved) version is the authoritative SoT evidence; superseded versions are reference-only.
승인본만 유효 — 현행(최신 승인) 버전만 정식 SoT 증빙으로 인정, 대체된 버전은 참조용.

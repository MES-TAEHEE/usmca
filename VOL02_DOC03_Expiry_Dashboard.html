01
Overview
화면 개요
The Expiry Dashboard is the highest-ROI, mission-critical screen of the DOC module — its single goal is zero missed expiry. It buckets every certificate by remaining validity into four tiers (D-90 Notice / D-30 Warn / D-7 Urgent / Expired), pushes tiered notifications to the document owner and procurement, tracks each alert's acknowledgement, and triggers supplier renewal requests routed to DOC-06. A daily scheduled scan re-buckets all certificates and a real-time recalculation fires on every new registration (BR-DOC-03). When a certificate expires it is flagged Expired and blocks label issuance downstream (BR-DOC-04), so the dashboard is the operational front line that prevents non-compliant shipments.
만료 대시보드는 DOC 모듈에서 ROI가 가장 높은 핵심 화면으로, 목표는 단 하나 — 만료 누락 ZERO입니다. 모든 증빙을 잔여 유효기간 기준 4단계(D-90 안내 / D-30 경고 / D-7 긴급 / 만료)로 분류하고, 담당자·구매팀에 단계별 알림을 발송하며, 알림 확인(Ack)을 추적하고, 공급사 갱신 요청을 DOC-06으로 라우팅합니다. 매일 예약 스캔으로 전 증빙을 재분류하고, 신규 등록 시 실시간 재계산이 동작합니다(BR-DOC-03). 만료된 증빙은 만료 표시되어 하위 라벨 발행을 차단(BR-DOC-04)하므로, 이 대시보드는 비적합 출하를 막는 운영 최전선입니다.

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-DOC-03
Tiered expiry alerts — daily scan buckets certificates into D-90 / D-30 / D-7 / Expired and pushes tiered notifications to owner + procurement, with acknowledgement tracking.
일일 스캔으로 증빙을 D-90/30/7/만료 티어로 분류, 담당자·구매팀에 단계별 알림 발송 및 확인(Ack) 추적.
자동
BR-DOC-04
Expired blocks label — once a certificate expires it is flagged Expired and any label issuance referencing it is blocked until a renewed version is registered.
만료 증빙은 만료 표시되며, 해당 증빙 참조 라벨 발행은 갱신 버전 등록 전까지 차단.
차단
BR-DOC-06
Audit trail — every alert send, acknowledgement, reminder, and export is logged (who / when / what) for audit retention.
알림 발송·확인·리마인드·내보내기 전수 로그 기록(감사 보존).

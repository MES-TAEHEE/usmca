01
Overview
화면 개요
The Origin Change History is an immutable audit log of every origin change made anywhere in the system — capturing who · when · before → after · reason for each edit (BR-ORG-02). Whenever a part's determined origin changes in ORG-01, the system writes a permanent record here that cannot be edited or deleted. Each change references the new supporting evidence (certificate / affidavit) that justified it (BR-ORG-01), so an auditor can trace any value back to its source document. This screen satisfies CBP and OEM auditability requirements and feeds the AUD module as the primary change-trail source. Compliance and Audit staff filter by part, period, changer, or country, inspect a before/after diff for any row, and export the trail for an audit response.
원산지 변경 이력은 시스템 전체에서 발생한 모든 원산지 변경의 불변(immutable) 감사 로그입니다. 각 변경의 누가·언제·이전→이후·사유를 기록합니다(BR-ORG-02). ORG-01에서 품목 원산지가 변경될 때마다 수정·삭제 불가한 영구 레코드가 이곳에 기록됩니다. 각 변경은 그 근거가 된 신규 증빙(인증서·진술서)을 참조하므로(BR-ORG-01), 감사자는 어떤 값이든 원천 문서까지 역추적할 수 있습니다. CBP·OEM의 감사 추적성 요건을 충족하며 AUD 모듈에 변경 추적 원천으로 공급됩니다. 컴플라이언스·감사 담당자는 품목·기간·변경자·국가로 조회하고, 행별 전/후 차이를 확인하며, 감사 대응용으로 내보냅니다.

Event & Action Flow

1
SYSTEMOrigin changed in ORG-01
사용자가 ORG-01에서 품목 원산지·판정결과 등을 변경하고 사유·근거 증빙을 입력 후 저장.
사유 미입력 시 ORG-01 저장 자체가 차단됨(BR-ORG-02).

시스템이 org_origin_change에 append-only 레코드 기록 — 누가·언제·이전→이후·사유·근거증빙 ID.
레코드는 생성 후 어떤 경로로도 수정·삭제 불가(불변).

✓

전 과정이 불변 로그로 보존되어 감사 추적성 충족.
DONEAuditable change trail delivered · 감사 추적 완료
05
Exception Handling
예외 처리
EXC-01Reason Required at Change변경 사유 필수
TriggerORG-01에서 원산지 변경 시 사유 미입력 (BR-ORG-02).
Action변경 저장 차단. 사유 입력 강제 → 미입력 시 이력 생성 안 됨.
변경 사유는 필수입니다 · Reason is required
EXC-02Delete Prohibited삭제 금지
Trigger사용자·관리자가 이력 레코드 삭제·수정 시도.
Action전면 차단. 불변(append-only) 정책. 시도 자체를 감사 로그에 기록.
변경 이력은 삭제·수정할 수 없습니다 (Immutable)
EXC-03No Evidence on Change근거 증빙 없음
Trigger원산지 변경에 근거 증빙이 연결되지 않음 (BR-ORG-01).
Action변경은 기록하되 ⚠ 경고 플래그. 감사 대상으로 표시·후속 보완 요청.
근거 증빙이 없습니다 — 보완 권고
EXC-04Large Export대량 내보내기
Trigger내보내기 대상 행이 임계(예: 50,000) 초과.
Action비동기 생성 → 완료 시 알림·다운로드 링크. 기간 분할 안내.
대량 데이터 — 백그라운드 생성 후 알림드립니다
EXC-05Concurrent Change Conflict동시 변경 충돌
Trigger동일 품목을 두 사용자가 동시에 변경 (ORG-01 단계).
Action낙관적 잠금 — 후행 저장 거부·재조회 요청. 두 시도 모두 이력 추적.
다른 사용자가 먼저 변경했습니다 — 새로고침 후 재시도
EXC-06No Permission권한 없음
TriggerORG.View 미보유 사용자 접근.
Action화면 접근 차단. 권한 신청 안내. (편집 권한은 본 화면과 무관 — 읽기 전용)
조회 권한이 없습니다 (ORG.View 필요)

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-ORG-02
Automatic immutable change log — every origin change auto-records who / when / before → after / reason; records are append-only and cannot be edited or deleted.
원산지 변경 시 누가·언제·이전→이후·사유를 자동 기록. append-only로 수정·삭제 불가.
자동
BR-ORG-01
Evidence on change — every origin change must reference the supporting evidence (certificate / affidavit) that justified it; missing evidence is flagged for audit follow-up.
모든 원산지 변경은 근거 증빙을 참조해야 함. 미연결 변경은 감사 보완 대상으로 표시.
경고
BR-ORG-05
Audit retention & export — the change trail is retained for the regulatory retention period and is exportable (Excel / PDF) as the primary source for CBP / OEM audit response; feeds the AUD module.
변경 이력은 법정 보존기간 동안 보관되며, CBP·OEM 감사 응답의 1차 원천으로 내보내기(Excel/PDF) 가능. AUD 모듈에 공급.

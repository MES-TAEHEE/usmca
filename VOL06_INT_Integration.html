INTEGRATION
시스템 연계 — SAP/ERP 연계 · BOM·공급사 마스터 동기화 · 데이터 정합성 검증 · 멀티-FTA 확장 기반
The Phase 4 enterprise/automation layer. Connects SAP/ERP and SCM systems to the UOCM platform, synchronizes BOM and Supplier Master data, validates data integrity before it propagates downstream, and provides the foundation for multi-FTA expansion. Once standard data is in place via automated sync, this layer powers ORG (BOM rollup, RVC calculation) and keeps MD master data continuously aligned with the system of record.
Phase 4 전사/자동화 계층. SAP/ERP·SCM 시스템을 UOCM 플랫폼과 연계하여 BOM·공급사 마스터 데이터를 동기화하고, 다운스트림 반영 전 데이터 정합성을 검증하며, 멀티-FTA 확장 기반을 제공합니다. 자동 동기화로 표준 데이터가 갖춰지면 이 계층이 ORG(BOM 롤업, RVC 계산)를 구동하고 MD 마스터 데이터를 원천 시스템과 지속 정합 상태로 유지합니다.

INT-02
BOM Sync
BOM 동기화 — SAP 자재구성 → ORG 롤업 기반
🖥 Web PCPhase 4
📋 L3 Detail →
👀
What you'll see: BOM 동기화 작업 현황 + 신규/변경/삭제 항목 비교 + 적용 결과 + 오류 내역SAP 자재구성을 UOCM BOM으로 자동 반영 — ORG RVC 계산 토대
EN · Description
Synchronizes BOM (bill of materials) structures from SAP into the UOCM platform. Detects added/changed/removed components, stages a diff for review, and applies validated changes. This is the data foundation that powers ORG's BOM rollup and RVC calculation once standard data is in place.
KO · 설명
SAP의 BOM(자재명세서) 구성을 UOCM 플랫폼으로 동기화. 추가/변경/삭제 부품을 감지하고 검토용 차이(diff)를 준비, 검증된 변경만 반영. 표준 데이터가 갖춰지면 ORG의 BOM 롤업·RVC 계산을 구동하는 데이터 토대.

EN · Description
Synchronizes Supplier Master records from SAP/SCM into the unified UOCM master. When the same supplier differs across sources, source-priority rules resolve the conflict deterministically. Keeps MD supplier master continuously aligned with the system of record and flags unresolved conflicts for review.
KO · 설명
SAP/SCM의 공급사 마스터를 통합 UOCM 마스터로 동기화. 동일 공급사가 소스 간 상이할 경우 소스 우선순위 규칙으로 충돌을 결정적으로 해소. MD 공급사 마스터를 원천 시스템과 지속 정합 상태로 유지하고 미해소 충돌은 검토 대상으로 표시.

INT-04 · ⚙ CORE
Data Integrity Monitor
데이터 정합성 검증 — 오류 격리·반영 보류 핵심 화면
🖥 Web PC⚙ Core
📋 L3 Detail →
👀
What you'll see: 정합성 검증 대시보드 + 오류 격리 큐 + 스키마 변경 경고 + 연계 로그·재처리잘못된 데이터가 ORG/MD로 흘러가기 전 차단하는 게이트키퍼
EN · Description
The gatekeeper screen. Validates every synced record before it propagates to ORG/MD; integrity errors are quarantined and held from downstream application (block). Detects source schema changes and warns operators, preserves a full integration log for audit, and supports re-validation/re-processing of held records.
KO · 설명
게이트키퍼 화면. 동기화된 모든 레코드를 ORG/MD 반영 전 검증; 정합성 오류는 격리되어 다운스트림 반영이 보류(차단)됩니다. 소스 스키마 변경을 감지·경고하고, 감사용 전체 연계 로그를 보존하며, 보류 레코드의 재검증·재처리를 지원.

01
Overview
화면 개요
The Supplier Master Sync screen pulls supplier master data from upstream SAP / SCM systems into UOCM so that DOC (certificates) and ORG (origin) modules always reference one consistent set of supplier records. Each run pulls suppliers, diffs them against the local md_supplier table, and resolves field-level conflicts using source-priority rules (BR-INT-03) — when SAP and UOCM disagree on a field, the configured priority source wins. Runs are transactional: any failure mid-apply triggers a full rollback (BR-INT-01) so the master is never left half-updated. Operators can trigger a sync on demand or rely on the daily batch, review counts and conflicts per run, and audit every applied change.
공급사 마스터 동기화는 상위 SAP / SCM 시스템의 공급사 마스터를 UOCM으로 끌어와, DOC(증빙)·ORG(원산지) 모듈이 항상 일관된 공급사 데이터를 참조하도록 합니다. 매 실행 시 공급사를 수집하고 로컬 md_supplier와 차이를 비교한 뒤, 소스 우선순위 규칙(BR-INT-03)으로 항목별 충돌을 해소합니다 — SAP와 UOCM 값이 다르면 설정된 우선 소스가 채택됩니다. 모든 실행은 트랜잭션이며, 적용 중 실패 시 전체 롤백(BR-INT-01)으로 마스터가 반쪽 갱신되지 않습니다. 담당자는 온디맨드 또는 일배치로 실행하고, 실행별 건수·충돌을 검토하며, 적용 변경을 전수 감사합니다.

Business Rules
업무 규칙
IDRule · 규칙Action
BR-INT-03
Source-priority resolution — when SAP and UOCM disagree on a field, the configured priority source (per field mapping in INT-02) is applied automatically; out-of-policy fields queue for manual review.
SAP·UOCM 값 불일치 시 항목별 소스 우선순위(INT-02 매핑)에 따라 자동 채택, 정책 외 항목은 수동 검토.
자동
BR-INT-01
Transactional rollback — sync applies as a single transaction; any failure mid-apply rolls back all changes so the master is never left partially updated.
동기화는 단일 트랜잭션. 적용 중 실패 시 전체 롤백 — 마스터 반쪽 갱신 금지.
차단
BR-INT-06
PII masking — personal data (contact names, phone numbers) pulled from source is masked per policy before storage; raw values are not retained.
원본의 개인정보(담당자명·연락처 등)는 저장 전 정책에 따라 마스킹, 원본 미보관.
자동

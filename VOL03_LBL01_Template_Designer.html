01
Overview
화면 개요
The Label Template Designer lets Label Admins design and manage reusable label templates for each OEM form — the COO standard layout plus customer-specific forms (현대 / 기아 / MOBIS). A WYSIWYG canvas defines field layout, required fields, barcode/QR placement, and paper size/orientation, drawing fields from a draggable palette (원산지국, 품목코드, LOT, 인증서번호, QR). Each OEM enforces its own required-field set (BR-LBL-05), and every template must pass approval before it can be used for issuance (BR-LBL-02). Approved templates become the layout source for LBL-02 (COO label issue) and LBL-03 (customer-form labels), eliminating ad-hoc, per-shipment label design.
라벨 템플릿 디자이너는 라벨 관리자가 OEM 양식별 재사용 가능한 라벨 템플릿을 설계·관리하는 화면입니다. COO 표준 레이아웃과 고객 전용 양식(현대/기아/MOBIS)을 지원합니다. WYSIWYG 캔버스에서 필드 배치·필수 필드·바코드/QR 위치·용지 크기/방향을 정의하며, 드래그 가능한 필드 팔레트(원산지국, 품목코드, LOT, 인증서번호, QR)에서 필드를 끌어 배치합니다. OEM별 필수 필드 세트가 강제되며(BR-LBL-05), 모든 템플릿은 사용 전 승인을 거쳐야 합니다(BR-LBL-02). 승인된 템플릿은 LBL-02·LBL-03 발행의 레이아웃 원본이 되어, 출하별 임의 라벨 설계를 제거합니다.

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-LBL-02
Approval before use — a template must be approved (via SYS routing) before it can be selected for issuance; draft / pending templates are not usable.
사용 전 승인 — 템플릿은 SYS 승인을 거쳐야 발행에 사용 가능. 초안·승인대기 템플릿은 사용 불가.
차단
BR-LBL-05
OEM required fields — each OEM (COO / 현대 / 기아 / MOBIS) enforces its own mandatory field set; the template cannot be submitted until all required fields are placed.
OEM 필수 필드 — OEM(COO/현대/기아/MOBIS)별 필수 필드 세트 강제. 모두 배치 전 승인 요청 불가.
차단
BR-LBL-06
Serial uniqueness (referenced) — barcode/QR fields encoding the issuance serial must reference the serial source enforced at issuance time, ensuring each printed label is uniquely identifiable.
시리얼 유일성(참조) — 발행 시리얼을 인코딩하는 바코드/QR 필드는 발행 시 강제되는 시리얼 소스를 참조해야 하며, 출력 라벨마다 고유 식별 보장.

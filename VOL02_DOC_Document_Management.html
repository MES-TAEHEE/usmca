DOCUMENT
MANAGEMENT
증빙 문서 관리 — Supplier Certificate 중앙 보관 · 만료 자동 알림 · 승인 워크플로우 · 버전 관리
The Phase 1 foundation. Establishes a single source of truth for all origin-supporting documents — supplier certificates of origin, USMCA certifications, affidavits, and HTS classifications. Centralizes scattered email/Excel evidence, auto-tracks certificate validity with tiered expiry alerts (D-90 / D-30 / D-7 / expired), enforces a multi-step approval/verification workflow, and keeps full version history for audit. Feeds AUD (traceability) and LBL (label issuance must reference a valid certificate).
Phase 1 토대. 모든 원산지 증빙 문서(공급사 원산지증명서, USMCA 인증, 진술서, HTS 분류)의 단일 진실원천(SoT)을 구축합니다. 이메일·엑셀에 분산된 증빙을 중앙화하고, 인증서 유효기간을 단계별 만료 알림(D-90 / D-30 / D-7 / 만료)으로 자동 추적, 다단계 승인·검증 워크플로우를 강제하며, 감사용 전체 버전 이력을 유지합니다. AUD(역추적)·LBL(라벨 발행 시 유효 증빙 참조)로 연계됩니다.

DOC-04
Approval Workflow
승인 워크플로우 — 다단계 결재·검증 라우팅
🖥 Web PCPhase 1
📋 L3 Detail →
👀
What you'll see: 결재함(상신/대기/승인/반려) + 검증 체크리스트 + 단계별 라우팅 + 코멘트증빙 적합성 검증 후에만 유효 처리
EN · Description
Multi-step approval/verification routing (Submitter → Reviewer → Approver). Each stage runs a compliance checklist (legibility, completeness, signatures, validity window). Reject returns to submitter with reasons. Only approved documents become "valid" in the SoT.
KO · 설명
다단계 승인·검증 라우팅(상신자→검토자→승인자). 각 단계에서 적합성 체크리스트(판독성·완전성·서명·유효기간) 수행. 반려 시 사유와 함께 상신자에게 반송. 승인된 문서만 SoT에서 "유효" 처리.

Version History
버전 이력 — 갱신·교체 추적 + 감사 로그
🖥 Web PCPhase 1
📋 L3 Detail →
👀

EN · Description

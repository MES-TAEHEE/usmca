01
Overview
화면 개요
Common Code Management is the central registry for shared reference codes consumed across every UOCM module — certificate types, origin-determination basis, the country list, HTS prefixes, alert tiers, and status enums. A two-pane layout lists code groups on the left and the selected group's items on the right, where each code carries KO/EN labels, a sort order, and an enable/disable toggle. Because these codes drive dropdowns, validation, and reporting everywhere, any change is impact-validated (BR-SYS-05) — disabling or removing an in-use code is blocked or warned with a "where used" list — and every definition change is logged (BR-SYS-02). Codes can be exported/imported for bulk maintenance and environment promotion.
공통 코드 관리는 전 UOCM 모듈이 공유하는 참조 코드(증빙 유형·원산지 판정 근거·국가 목록·HTS 접두·알림 등급·상태 enum)의 중앙 레지스트리입니다. 좌측에 코드 그룹, 우측에 선택 그룹의 코드 항목(KO/EN 명칭·정렬·사용여부 토글)을 표시합니다. 이 코드는 전 화면의 드롭다운·검증·리포트를 구동하므로 변경 시 영향도 검증(BR-SYS-05)을 수행 — 사용 중 코드 비활성·삭제는 "사용처" 목록과 함께 차단·경고됩니다. 모든 정의 변경은 로그(BR-SYS-02). 대량 유지보수·환경 이행을 위해 내보내기/가져오기를 지원합니다.

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-SYS-05
Impact validation — before disabling, deleting, or relabeling any code, the system runs a "where used" check; in-use codes are blocked or warned with the referencing modules/count.
코드 비활성·삭제·명칭 변경 전 "사용처" 검증. 사용 중 코드는 참조 모듈·건수와 함께 차단·경고.
차단
BR-SYS-02
Change logged — every code definition change (create / edit / enable-disable / import) is recorded with actor, timestamp, before/after for audit.
코드 정의 변경(생성·수정·사용여부·가져오기) 전수를 행위자·시각·전후값과 함께 감사 로그 기록.
자동
BR-SYS-01
Admin-only authority — code definition and management requires the SYS.Admin role; SYS.View users have read-only access.
코드 정의·관리는 SYS.Admin 권한 필요. SYS.View 사용자는 열람만 가능.

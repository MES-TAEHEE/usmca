01
Overview
화면 개요
The Certificate Registry is the single source of truth for every origin-supporting document. A searchable grid lists all certificates with status color-coding (valid / expiring / expired), and a slide-in drawer registers new documents via drag-and-drop upload + mandatory metadata. On save, the system validates required fields (BR-DOC-01), schedules tiered expiry alerts (BR-DOC-03), sets status to Pending Review until approval (BR-DOC-02), and links the certificate to supplier / part / BOM entities for downstream traceability. This screen replaces the scattered email–Excel–shared-drive workflow that today has no single source of truth.
증빙 등록부는 모든 원산지 증빙의 단일 진실원천(SoT)입니다. 검색 가능한 그리드가 전 증빙을 상태별 색상(유효/임박/만료)으로 표시하며, 슬라이드 드로어에서 드래그앤드롭 업로드 + 필수 메타데이터로 신규 등록합니다. 저장 시 필수항목 검증(BR-DOC-01), 단계별 만료 알림 예약(BR-DOC-03), 승인 전까지 검토중 상태 설정(BR-DOC-02), 공급사·품목·BOM 연결(역추적 기반)을 수행합니다. 이메일·엑셀·공유폴더에 분산된 현행 방식을 대체합니다.

Exception Handling
예외 처리
EXC-01Missing Required Field필수항목 누락
Trigger저장 시 필수 메타데이터 누락 (BR-DOC-01).
Action저장 차단. 누락 필드 빨강 테두리 + 포커스 이동.
필수 항목을 입력하세요 · Required fields missing
EXC-02Duplicate Cert No.인증서번호 중복
Trigger기존 인증서번호와 동일 번호 등록 시도.
Action경고 후 "신규 버전으로 등록" 옵션 제시 (BR-DOC-05).
동일 번호 존재 — 버전 갱신으로 처리하시겠습니까?
EXC-03Invalid Date Range유효기간 오류
Trigger만료일 < 발급일.
Action인라인 검증 실패. 저장 비활성.
만료일은 발급일 이후여야 합니다
EXC-04Unregistered Supplier미등록 공급사
TriggerMD에 없는 공급사 선택 시도.
Action선택 불가. MD Supplier 선등록 안내 링크 제공.
공급사를 기준정보(MD)에 먼저 등록하세요
EXC-05File Too Large / Type파일 용량·형식
Trigger20MB 초과 또는 비허용 형식.
Action업로드 거부. 허용 형식·용량 안내.
PDF/JPG/PNG · 20MB 이하만 업로드 가능
EXC-06No Edit Permission권한 없음
TriggerDOC.View만 보유한 사용자.
Action등록 버튼 비활성. 열람만 가능.
등록 권한이 없습니다 (DOC.Edit 필요)

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-DOC-01
Mandatory metadata — cert no., supplier, origin country, type, issue/expiry date, applied parts (≥1) required before save.
등록 시 인증서번호·공급사·원산지국·유형·발급/만료일·적용품목(1건+) 필수.
차단
BR-DOC-02
Approval before valid — document stays "Pending Review" until DOC-04 approval; not counted as valid SoT evidence until then.
승인 전까지 "검토중", 승인 후에만 유효 증빙으로 인정.
차단
BR-DOC-03
Tiered expiry alerts — on save, schedule D-90 / D-30 / D-7 / expired notifications to owner + procurement.
저장 시 D-90/30/7/만료 알림 자동 예약 (담당자·구매팀).
자동
BR-DOC-05
Immutable versioning — approved certificates cannot be deleted; renewals create a new version, prior versions preserved.
승인 증빙 삭제 불가. 갱신은 신규 버전 생성, 이전 버전 보존.
차단
BR-DOC-06
Audit trail — every view, download, and status change is logged (who / when / what) for audit retention.
조회·다운로드·상태변경 전수 로그 기록(감사 보존).
자동

01
Overview
화면 개요
The System Audit Trail is the immutable, tamper-evident record of every security-relevant event across the platform — login / logout, access attempts, permission changes, and sensitive data view / edit / export (BR-SYS-02). A searchable grid lets Security and Auditor roles filter by user, event type, module, period, and IP, with a row detail panel exposing the full event payload (before/after values, session, user-agent). Entries are append-only: they can never be edited or deleted, and even read access to this log is itself logged. This is distinct from the business audit module (AUD) which supports customs/origin audits — SYS-02 is the system & security audit that proves who did what, when, and from where.
시스템 감사 로그는 플랫폼 전반의 모든 보안 관련 이벤트(로그인·로그아웃·접근·권한 변경·민감정보 조회/변경/내보내기)를 불변·위변조 탐지 가능한 형태로 기록합니다(BR-SYS-02). 보안·감사 담당자는 사용자·이벤트 유형·모듈·기간·IP로 검색하고, 행 상세 패널에서 변경 전후값·세션·UA 등 전체 이벤트 페이로드를 확인합니다. 모든 항목은 추가 전용(append-only)으로 편집·삭제가 불가하며, 로그 자체에 대한 열람도 다시 로그로 남깁니다. 관세·원산지 감사를 지원하는 업무 감사(AUD)와 달리, 본 화면은 누가·언제·어디서 무엇을 했는지를 증명하는 시스템·보안 감사입니다.

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-SYS-02
Immutable security log — all security-relevant events are recorded append-only; entries cannot be edited or deleted, are tamper-evident, retained per policy, and access to the log itself is logged.
모든 보안 이벤트는 추가 전용으로 기록 — 편집·삭제 불가, 위변조 탐지, 정책에 따른 보존, 로그 열람도 기록.
차단
BR-SYS-06
Login & lockout — every login / logout is logged; N consecutive failures lock the account and raise a CRIT event to security.
로그인·로그아웃 전수 기록 — N회 연속 실패 시 계정 잠금 + 보안팀 CRIT 알림.
차단
BR-SYS-01
RBAC scope — only roles holding SYS.Audit may read the audit trail; the log records actor identity from the authenticated session for every event.
SYS.Audit 권한 보유자만 감사 로그 열람 가능 — 모든 이벤트는 인증 세션의 행위자 식별 정보를 함께 기록.

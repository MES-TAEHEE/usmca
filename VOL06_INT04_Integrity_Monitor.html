01
Overview
화면 개요
The Data Integrity Monitor continuously validates cross-system data integrity — detecting parts without BOM, BOM nodes without origin, certificates referencing unknown suppliers, orphan records, and sync drift across the integrated landscape (ERP / PLM / supplier portal). Each rule runs on a schedule; violations are isolated and surfaced as actionable issues (BR-INT-02) rather than allowed to silently corrupt downstream origin determination. A KPI strip summarizes total / critical / warning / resolved issues, a rules-and-issues grid lets users drill into each violation, and the issue-detail panel lists every affected record with a suggested fix and a jump-to-source link. This screen is the safety gate that keeps bad data from poisoning ORG, MD, and AUD.
데이터 정합성 검증 화면은 연계된 시스템(ERP·PLM·공급사 포털) 전반의 교차 정합성을 상시 검증합니다. BOM 미연계 부품, 원산지 없는 BOM 노드, 미등록 공급사를 참조하는 증빙, 고아 레코드, 동기화 드리프트 등을 탐지하며, 위반은 격리되어 실행 가능한 이슈(BR-INT-02)로 표면화됩니다 — 잘못된 데이터가 하류 원산지 판정을 오염시키지 못하도록 차단합니다. KPI 스트립(총 이슈·치명·경고·해결됨), 규칙·이슈 그리드, 이슈 상세 패널(영향 레코드·권장 조치·원본 이동)으로 구성됩니다.

05
Exception Handling
예외 처리
EXC-01Critical Blocks Downstream치명 오류 하류 차단
Trigger치명 정합성 위반 존재 (BR-INT-02).
Action해당 데이터 영역의 ORG 원산지 판정 차단. 차단 배너 + 미해결 카운트 표시.
치명 정합성 오류 — 하류 판정이 차단되었습니다
EXC-02False Positive Ignore거짓양성 무시
Trigger실제 문제가 아닌 위반으로 판단.
Action[무시]는 사유 필수 입력. 무시 이력 로그 기록(BR-INT-04).
무시 사유를 입력해야 처리됩니다
EXC-03Rule Misconfiguration규칙 오설정
Trigger규칙 정의 오류로 대량 거짓양성 발생.
Action규칙 비활성화 안내. 관리자에게 재설정 요청 링크.
규칙 설정 확인 필요 — 관리자에게 문의
EXC-04Large Issue Set대량 이슈
Trigger단일 규칙에서 수백 건 이상 위반 발생.
Action상세 목록 페이징 + 일괄 처리(Bulk Resolve) 제공.
대량 이슈 — 일괄 처리 모드를 사용하세요
EXC-05Re-check After Fix수정 후 재검사
Trigger원본 수정 후 위반 잔존 여부 확인.
Action[재검사]로 해당 규칙만 즉시 재평가. 소멸 시 자동 해결.
재검사 결과 위반 없음 — 해결 처리됨
EXC-06No Permission권한 없음
TriggerINT.View만 보유한 사용자.
Action[해결]·[무시] 비활성. 조회만 가능.
해결 권한이 없습니다 (INT.Resolve 필요)

07
Business Rules
업무 규칙
IDRule · 규칙Action
BR-INT-02
Isolate errors — integrity violations are isolated and surfaced as actionable issues; critical errors block downstream origin determination until resolved, preventing bad data from corrupting results.
정합성 오류는 격리·이슈화. 치명 오류는 해결 전까지 하류 원산지 판정을 차단하여 잘못된 데이터의 오염을 방지.
차단
BR-INT-04
Audit logging — every check run, violation, severity change, resolve, and ignore-with-reason is logged (who / when / what) for audit retention.
검사 실행·위반·심각도 변경·해결·사유무시를 전수 로그 기록(누가/언제/무엇, 감사 보존).
자동
BR-INT-05
Schema change detection — when an integrated source's schema changes, affected rules are flagged and re-validated to catch new drift or broken mappings.
연계 소스 스키마 변경 감지 시 영향 규칙을 표시·재검증하여 신규 드리프트·매핑 오류를 포착.

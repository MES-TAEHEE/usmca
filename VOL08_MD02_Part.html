01
Entity Overview
엔티티 개요
The Part / Item entity is the central material master for the entire compliance system — covering raw materials, components, sub-assemblies, and finished goods. Every part carries a stable part_code, a default declared origin country, an HTS classification, and an optional sourcing supplier. It is the anchor entity that origin determination (ORG), BOM explosion (MD-03), HTS classification (MD-05), and certificates / labels (DOC, LBL) all reference. Parts may be created manually by Engineering or synchronized from SAP Material Master via the INT integration layer, keeping a single canonical item list across MES and ERP.
품목(Part / Item) 엔티티는 전 컴플라이언스 시스템의 핵심 자재 마스터로, 원자재·부품·반제품·완제품을 모두 포괄합니다. 모든 품목은 고정 part_code, 기본 신고 원산지국, HTS 분류, 선택적 소싱 공급사를 보유합니다. 원산지 판정(ORG), BOM 전개(MD-03), HTS 분류(MD-05), 증빙·라벨(DOC, LBL)이 모두 참조하는 앵커 엔티티입니다. Engineering이 수기로 생성하거나 INT 연계로 SAP 자재마스터에서 동기화되어 MES·ERP 전반의 단일 품목 목록을 유지합니다.

06
Business Rules
업무 규칙
IDRule · 규칙Action
BR-MD-01
Unique part code — part_code must be unique system-wide and is immutable once referenced.
품번(part_code)은 시스템 전체에서 유일하며, 참조 발생 후 변경 불가.
차단
BR-MD-02
Mandatory classification — every part must carry an HTS code and default origin country before use in ORG / certificates.
ORG·증빙 사용 전 모든 품목은 HTS 코드·기본 원산지국을 보유해야 함.
경고
BR-MD-04
Finished-good origin roll-up — a finished part's declared origin must be derived via BOM origin roll-up, not set manually (link ORG-04).
완제품 원산지는 수기 입력이 아닌 BOM 롤업으로 산정해야 함 (ORG-04 연계).
자동
BR-MD-05
Soft delete only — referenced parts cannot be hard-deleted; set status='inactive' instead to preserve traceability.
참조된 품목은 물리 삭제 불가. status='inactive' 처리로 역추적성 보존.
차단
BR-MD-06
In-house parts — finished / sub-assembly parts may have NULL supplier; raw / component parts should reference a sourcing supplier.
완제품·반제품은 공급사 NULL 허용. 원자재·부품은 소싱 공급사 지정 권장.

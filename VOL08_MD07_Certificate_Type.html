01
Entity Overview
엔티티 개요
The Certificate Type entity defines the controlled set of evidence types used across the DOC module — COO, USMCA, AFFID, HTS, MILL, and OEMDOC. Each type carries its own expiry policy (validity months — null means perpetual), approval requirement, and blanket-certification rule. These attributes drive runtime behavior in DOC-01 (Certificate Registry — type dropdown, auto-applied expiry policy) and DOC-03 (Expiry Dashboard — tiered alert scheduling). Because it is a code-type master, rows are seeded once and rarely change; new evidence types are added here before they can be selected anywhere in DOC.
증빙 유형 엔티티는 DOC 모듈 전반에서 사용되는 증빙 종류 집합(COO·USMCA·AFFID·HTS·MILL·OEMDOC)을 정의합니다. 각 유형은 고유의 만료 정책(유효개월 — null=무기한), 승인 필요 여부, 블랭킷 인증 허용 규칙을 가집니다. 이 속성들이 DOC-01(증빙 등록 — 유형 선택·만료 정책 자동 적용)과 DOC-03(만료 대시보드 — 단계별 알림 예약) 동작을 제어합니다. 코드성 기준정보이므로 1회 시딩 후 거의 변경되지 않으며, 신규 증빙 유형은 DOC에서 선택되기 전 이곳에 먼저 등록합니다.

06
Business Rules
업무 규칙
IDRule · 규칙Action
BR-MD-01
Code-type master is seeded once; cert_type_code must be unique and is immutable after first reference by DOC.
코드성 기준정보는 1회 시딩 · cert_type_code 유니크 필수, DOC 최초 참조 후 변경 불가.
차단
BR-MD-02
Soft-deactivate only — a type referenced by any certificate cannot be hard-deleted; set is_active=0 to retire.
참조된 유형은 물리 삭제 불가 · is_active=0 으로만 비활성화.
차단
BR-MD-05
All masters carry created/updated audit columns; every edit logs who/when (MD.Edit RBAC enforced).
전 기준정보 생성·수정 감사 컬럼 보유 · 수정 시 작업자/일시 기록(MD.Edit 권한).
자동
BR-DOC-03
validity_months + default_alert_tiers feed DOC tiered expiry alerts (D-90/30/7); NULL validity = no expiry alert.
validity_months·default_alert_tiers → DOC 단계별 만료 알림(D-90/30/7); NULL=만료 알림 없음.
자동
BR-MD-07
HTS type allows perpetual validity — validity_months = NULL is valid and skips expiry scheduling entirely.
HTS 유형은 무기한 허용 — validity_months = NULL 유효, 만료 예약 미수행.
